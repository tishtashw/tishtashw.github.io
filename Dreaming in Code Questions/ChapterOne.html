Dreaming in Code Questions

Chapter 1

  1. What roles in the Chandler project did Michael Toy, John Anderson, Ted Burgess, Mitchell Kapor, and Lou Montulli hold. Michael Toy was the manager, John Anderson was the systems architect, Jed Burgess was one of the projects young programmers,Mitchell Kapor was/is the found the funder of the open source applications foundation, and Lou Montulli was another Chandler programmer.
  2. What is “Bugzilla?” A program which holds the list of live bugs that must be fixed in their product a.k.a Chandler
  3. What is OSAF? Open Source Applications Foundation
  4. What is the projects name? Chandler
  5. What will the software do? Chandler is supossed to be a PIM or a “personal information manager”, basically organizing and sharing calendars, to-do lists, and other information
  6. What is Toy’s keyword for “black hole” bugs? “Scary”
  7. What scared Toy so much about Bug 44? The fact that no one can guess how long it will take to fix
  8. What did Toy refer to as a “snake?” An important problem that no one knew how to attack
  9. In the software world, what does “slippage” mean? Falling behind schedule
  10. Fredrick Brooks was a programming manager for what software project? IBM, IBM System/360 specifically
  11. What is Brooks's Law? “Adding manpower to a late software project makes it later.”
  12. Brooks found what % of project time was spent writing code? 1/6th of the time is spent writing code
  13. Brooks found what % of project time was for testing and fixing bugs? 50%, or fully half the time is spent testing and fixing bugs
  14. Brooks observed that the unit of effort named “man-month” only applied under what conditions? With many workers and no communication among them
  15. What is the difference between source code and the program you install (.exe) on your computer? The source code allows for modifications while the exe only allows the program to run on the system it’s downloaded on
  16. What is the one “article of faith” that all “open source” or “free” software advocates share? That software anyone can tinker with is bound to improve over time in ways that “closed” software can’t match
  17. What is the difference between a “good” programmer and a “great” programmer? A “good” programmer knows what to write but a “great” programmer knows what to reuse and rewrite
  18. Eric Raymond’s book “The Cathedral and the Bazaar” made a distinction between two important project development ideas, briefly contrast them. The idea is that some software to be sucessful needed built like a cathedral, carefully crafted by individual wizards or small bands of mages working in isolation. Others later found that the method that more closely resembled a bazaar was effective, the idea being that the more people looking at a problem the quicker it was solved.
  19. Has “open source” software project development refuted Brooks’s “mythical man-month” concerns? I don’t know that it has completely refuted Brook’s concerns but simply proven that there is a way for lots of people to work on a project and still have it be effective.
  20. What was Andy Hertzfeld’s input when the Chandler project appeared to have stalled? His suggestion was to get something working quick, so that it can be improved upon rather than spinning in place trying to get something perfect working
